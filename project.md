# Калькулятор аутсорсера — Документация проекта

## Что это за проект

**Калькулятор стоимости часа персонала для аутсорсинговой компании** — веб-приложение для финансистов и руководителей. Позволяет мгновенно рассчитать, сколько стоит один час работы сотрудника для заказчика, исходя из себестоимости, накладных расходов и желаемой маржинальности.

**Создан:** финансистами с опытом в аутсорсинге 4+ лет
**Авторы:** Екатерина Яхонтова ([@yahontova_finance](https://t.me/yahontova_finance)) и Елена Демина ([@elenademina_findoctor](https://t.me/elenademina_findoctor))
**Репозиторий:** https://github.com/3741731-ekaterina/kalrylyator-autsorsera

## Математическая модель

Ключевая идея: **цена для заказчика (Price) — это неизвестное X**. Накладные расходы и прибыль считаются не от себестоимости, а от самой итоговой цены.

### Формула

```
Price = (S_worker + S_smz + S_manager + M_head)
        ─────────────────────────────────────────
        1 − (C_overhead / 100) − (P_profit / 100)
```

### Расшифровка переменных

| Переменная | Обозначение | Описание |
|---|---|---|
| `S_worker` | ЗП рабочего | Базовая ставка, руб/час |
| `S_smz` | СМЗ-налог | `S_worker × 0.06`, если включен |
| `S_manager` | ЗП менеджера | `S_worker × (M_manager / 100)` |
| `M_head` | ЗП руководителя | Фиксированная сумма, руб/час |
| `C_overhead` | Накладные расходы | % от итоговой цены (офис, налоги) |
| `P_profit` | Целевая прибыль | % маржинальности от итоговой цены |

### Производные показатели

```
Стоимость смены       = Price × H_shift
Валовая прибыль смены = Стоимость смены × (P_profit / 100)
```

### Защита от деления на ноль

Если `C_overhead + P_profit ≥ 100%`, делитель становится ≤ 0 — в этом случае цена возвращается как 0.

## Входные параметры (ввод пользователя)

Все параметры имеют **слайдер + числовое поле**, реагируют мгновенно.

| Параметр | Тип | Диапазон | По умолчанию | Описание |
|---|---|---|---|---|
| ЗП рабочего | руб/час | 100–2000 | 450 | Базовая ставка сотрудника |
| Налог СМЗ | чекбокс/toggle | вкл/выкл | вкл | Добавляет 6% к ЗП рабочего |
| Накладные расходы | % | 0–50 | 11 | Офис, бухгалтерия, налоги компании |
| ЗП менеджера | % от ЗП | 0–100 | 10 | Процент от базовой ставки рабочего |
| ЗП руководителя | руб/час | 0–500 | 25 | Фиксированная сумма в каждый час |
| Целевая прибыль | % | 0–50 | 20 | Желаемая маржинальность |
| Часов в смене | ч | 1–24 | 8 | Для расчёта стоимости смены |

## Выходные данные (результаты)

### Блок А — Итоговые показатели (3 карточки)

| Показатель | Формула |
|---|---|
| Цена для заказчика | `Price`, руб/час |
| Стоимость одной смены | `Price × H_shift`, руб |
| Прибыль со смены | `Смена × (P_profit / 100)`, руб |

### Блок Б — Детализация расходов

Отображается в виде списка с цветными прогресс-барами и пончик-диаграммой:

| Статья | Формула | Цвет |
|---|---|---|
| Выплаты рабочему | `S_worker + S_smz` | Фиолетовый |
| Налоги и офис | `Price × C_overhead / 100` | Оранжевый |
| ЗП персонала (менеджер + руководитель) | `S_manager + M_head` | Зелёный |
| Чистая прибыль компании | `Price × P_profit / 100` | Розовый |

Каждая строка показывает: **сумму в ₽/ч**, **% от цены** и **цветную полосу-заполнение**.

## Компоненты

### `App.tsx` — главный компонент

Управляет всем состоянием приложения:

- `inputs` — объект со всеми параметрами калькулятора
- `savedCalculations` — история сохранённых расчётов (хранится в `localStorage`)
- `result` — вычисляется через `useMemo` при каждом изменении `inputs`

**Основные действия:**
- `handleSave()` — сохранить текущий расчёт с произвольным названием
- `handleDelete(id)` — удалить расчёт из истории
- `loadCalculation(saved)` — загрузить ранее сохранённый расчёт

### `InputControl.tsx` — слайдер с числовым полем

**Props:** `label`, `value`, `onChange`, `min`, `max`, `step`, `unit`, `hint`

Отображает: метку + иконку подсказки (i) + числовой инпут + единицу измерения + слайдер с заполненным треком (фиолетовый до ползунка, серый после).

### `BreakdownChart.tsx` — диаграмма структуры цены

**Props:** `data: CalculationResult`, `centerValue: number`

Рендерит:
- **Левая часть:** список из 4 статей расходов с цветным кружком, названием, суммой, процентом и прогресс-баром
- **Правая часть:** пончик-диаграмма (Recharts `PieChart` с `innerRadius`) с кастомным тёмным тултипом и центральным текстом "Итого / ₽/ЧАС"

### `utils/calculations.ts` — логика расчёта

Чистая функция `calculateOutsourcingRate(inputs) → CalculationResult`. Никакого side-эффекта, никакой зависимости от UI.

## Типы данных (`types.ts`)

```typescript
interface CalculatorInputs {
  workerSalary: number;      // ЗП рабочего, руб/час
  smzTaxEnabled: boolean;    // включён ли налог СМЗ
  overheadPercent: number;   // накладные расходы, %
  managerPercent: number;    // ЗП менеджера, % от ЗП рабочего
  deptHeadSalary: number;    // ЗП руководителя, руб/час
  profitPercent: number;     // желаемая прибыль, %
  shiftHours: number;        // длительность смены, ч
}

interface CalculationResult {
  pricePerHour: number;          // цена для заказчика
  pricePerShift: number;         // стоимость смены
  grossProfitPerShift: number;   // прибыль со смены
  breakdown: {
    workerTotal: number;    overhead: number;
    staffTotal: number;     netProfit: number;
    workerPercent: number;  overheadPercent: number;
    staffPercent: number;   profitPercent: number;
  };
}

interface SavedCalculation {
  id: string;          // UUID
  name: string;        // пользовательское название
  inputs: CalculatorInputs;
  result: CalculationResult;
  timestamp: number;   // Date.now()
}
```

## Функция «Сохранить в историю»

Пользователь может сохранить текущий расчёт с произвольным названием (например, «Склад А», «Ритейл», «Проект Х»). Все расчёты хранятся в `localStorage` под ключом `outsourcer_calcs` в формате JSON.

**Возможности:**
- Сохранить расчёт с именем
- Просмотреть список сохранённых (показывается цена и % прибыли)
- Загрузить любой сохранённый расчёт одним кликом (восстанавливает все параметры)
- Удалить расчёт из архива

## Деплой на GitHub Pages

### Проблема и решение

Исходный код содержит TypeScript-файлы (`.tsx`), которые браузер не умеет запускать напрямую. GitHub Pages отдаёт файлы как статику — поэтому без сборки открывалась пустая страница.

**Решение:** настроен автоматический деплой через GitHub Actions.

### Как работает деплой

При каждом `git push` в ветку `main`:

1. GitHub Actions запускает `ubuntu-latest`
2. Устанавливает Node.js 20 + зависимости (`npm install`)
3. Собирает проект: `npm run build` (Vite → папка `dist/`)
4. Загружает `dist/` как артефакт GitHub Pages
5. Публикует на `https://3741731-ekaterina.github.io/kalrylyator-autsorsera/`

### Base path

В `vite.config.ts` задан условный `base`:

```typescript
base: process.env.GITHUB_ACTIONS ? '/kalrylyator-autsorsera/' : '/'
```

Локально (`npm run dev`) — приложение доступно на `http://localhost:3000/`.
На GitHub Pages — на `https://3741731-ekaterina.github.io/kalrylyator-autsorsera/`.

## Адаптивность

| Ширина экрана | Лэйаут |
|---|---|
| < 900px (мобильный) | Всё в одну колонку: сначала параметры, затем результаты |
